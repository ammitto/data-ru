= Ammitto Russia Sanctions Data Repository

== Purpose

This repository stores downloaded and processed sanctions data from the
Russian Federation government sanctions announcements.

The data is maintained by the Ammitto project and used by the `ammitto` Ruby
gem for querying international sanctions information.

== Legal Basis

Russia maintains sanctions and restrictive measures lists through:

* **Стоп-лист (Stop-list)** - Ministry of Foreign Affairs (MID)
* **Central Bank Sanctions** - Central Bank of Russia (CBR)
* **Government Decrees** - Постановления

These are implemented through federal laws and government decrees
authorizing reciprocal sanctions against foreign persons and entities.

== Data Source

Russia maintains sanctions lists through:

.Stop-list (Стоп-лист)
* **Authority**: Ministry of Foreign Affairs (МИД России)
* **URL**: https://mid.ru
* **Effects**: Entry bans on foreign persons

.Central Bank Sanctions
* **Authority**: Central Bank of Russia (ЦБ РФ)
* **Effects**: Financial sanctions

.Government Decrees
* **Authority**: Government of Russia
* **Effects**: Various economic restrictions

== Special Considerations

* **Format**: HTML announcements (not structured XML/JSON)
* **Language**: Russian with English transliterations
* **Parsing**: Requires HTML extraction from official announcements
* **Encoding**: UTF-8 with Cyrillic characters

== Directory Descriptions

downloaded/:: Contains raw HTML pages downloaded from Russian government
websites. Data must be parsed from official announcements.

processed/:: Contains individual YAML files, one per sanction entity.
Files are named using the pattern `{primary-name}-{index}.yaml` where
index ensures uniqueness for entities with similar names.

reference-docs/:: Contains reference documentation and translations.

update.log:: Log file tracking when data was last updated with timestamps.

== Updating Data

Data is automatically updated daily via GitHub Actions.

To manually update using the ammitto gem:

[source,shell]
----
# From the ammitto gem directory
bundle exec ruby exe/ammitto fetch ru --format yaml --output-dir ./processed
----

Or via the adapters repository:

[source,shell]
----
# From the adapters/ directory
ruby adapter.rb
----

== Source Data Format

The processed YAML files in `processed/` follow this structure:

=== Person Entity
[source,yaml]
----
names:
  - "PRIMARY NAME (Кириллица)"
  - "PRIMARY NAME (Latin)"
entity_type: person
source: ru
country: Country Name
list_type: "stop-list"
sanction_measures:
  - "Entry ban"
----

=== Organization Entity
[source,yaml]
----
names:
  - "ORGANIZATION NAME"
entity_type: organization
source: ru
country: Country Name
list_type: "stop-list"
sanction_measures:
  - "Financial restrictions"
----

=== Field Definitions

[cols="1,4"]
|===
| Field | Description

| `names`
| Array of name variants (may include Cyrillic and Latin scripts)

| `entity_type`
| Entity type: person or organization

| `source`
| Source code: ru

| `country`
| Country associated with the entity

| `list_type`
| Type of Russian sanctions list: stop-list, central-bank, government

| `sanction_measures`
| Array of sanction measures applied

|===

== Harmonized Data Format

After harmonization, entities are transformed into JSON-LD format:

=== Entity (JSON-LD)
[source,json]
----
{
  "@context": "https://www.ammitto.org/schema/v1/context.jsonld",
  "@id": "https://www.ammitto.org/entity/ru/001",
  "@type": "PersonEntity",
  "names": [
    {
      "@type": "NameVariant",
      "fullName": "PRIMARY NAME (Кириллица)",
      "script": "Cyrl",
      "isPrimary": true
    },
    {
      "@type": "NameVariant",
      "fullName": "PRIMARY NAME (Latin)",
      "script": "Latn",
      "isPrimary": false
    }
  ],
  "sanctionEntries": [
    {
      "@id": "https://www.ammitto.org/entry/ru/001",
      "@type": "SanctionEntry",
      "authority": {
        "@id": "https://www.ammitto.org/authority/ru",
        "name": "Russia"
      },
      "regime": {
        "@type": "SanctionRegime",
        "code": "STOP_LIST",
        "name": "Stop List"
      },
      "effects": [
        {
          "@type": "SanctionEffect",
          "effectType": "travel_ban",
          "scope": "full"
        }
      ],
      "status": "active"
    }
  ]
}
----

== Entity Types

The Russian sanctions lists contain the following entity types:

Person:: Individual persons subject to sanctions
Organization:: Legal entities, companies, and organizations

== Knowledge Graph Integration

This repository's processed YAML data feeds into the Ammitto knowledge graph.

When the harmonization process runs:

[source,shell]
----
# Auto-detect all data-* repos and harmonize
ammitto harmonize --scan --sources-dir /path/to/repos --output-dir ./api/v1
----

The ru data is transformed into:

* Individual JSON-LD nodes in `api/v1/node/entity/ru/{id}.jsonld`
* Sanction entries in `api/v1/node/entry/ru/{id}.jsonld`
* Lightweight search index entries
* Facet counts for filtering
